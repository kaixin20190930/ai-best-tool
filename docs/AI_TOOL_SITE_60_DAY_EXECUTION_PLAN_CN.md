> 如果你需要看现在真正要做什么，请只看总控表。

这份文档不是战略宣言，是接下来 60 天的执行清单。

现在的判断很直接：

- 你已经有工具库、分类、指南、对比页、提交、支付和管理后台
- 当前最缺的，不是更多功能
- 当前最缺的，是验证“工具方愿意付费、愿意留资、愿意回话”

### 已完成的核心模块

- 工具库

- Guide 体系
- Comparison 体系
- Submit 流程
- Stripe 支付

### 当前已知规模

### 目前阶段判断

项目现在已经不是“继续做产品”的阶段，而是“验证商业模式”的阶段。

换句话说：

- 不要再用新增功能证明忙碌
- 要用付费、认领、回头购买证明价值
- 要用真实外联和真实留资证明工具方愿意回话

## 三、总体策略

### 1. 先把“为什么付费”讲清楚

用户付费不是因为页面多，而是因为他一眼能看懂：

- 我买的是更快审核
- 我买的是固定曝光窗口
- 我买的是更早出现在用户面前的机会
- 我不是在买永久广告位
- 我不是在为没有流量的展示位买单

### 2. 先做轻量认领，不做复杂 owner 系统

先验证有没有人愿意认领：

- 先收邮箱
- 先做 Landing Page
- 先确认需求强不强
- 再决定要不要做完整 dashboard

认领应该尽早开始，不要拖到后面。

如果工具方连邮箱都不愿意留，说明 owner 体系还不值得重投入。

### 3. 内容扩张必须服务转化

接下来新增的 Guide / Comparison / 分类页，都必须回答一个问题：

> 它能不能把用户推进到详情页、提交页或付款页？

如果不能，就暂缓。

## 四、第一阶段：未来 7 天

### 目标

验证是否有人愿意付费，顺手验证是否有人愿意留资。

### 核心指标

- 第一笔真实付费
- 第一笔 Priority Review 付费
- 第一批 Claim Listing 留资
- 第一批页面级埋点可读

### P0 任务

#### 1. Pricing 页面重构

目标：

- 让用户 3 秒理解为什么要付费

任务：

- 重写 Pricing 文案
- 删除无关描述
- 强调交付内容
- 强调审核时间
- 强调曝光周期
- 强调不保证流量和排名

输出结构：

- Free
- Priority Review
- Featured
- Priority + Featured

验收标准：

- 用户能快速看懂每个方案买的是什么
- 页面没有 roadmap、产品讨论、内部思考内容
- 页面重点是交易，不是叙事
- 页面里不要出现“先做什么、后做什么”这类内部规划口吻

#### 2. Submit 页面重构

目标：

- 提交前完成教育

任务：

- 增加套餐对比
- 增加审核流程说明
- 增加 FAQ
- 增加审核标准
- 增加拒绝规则
- 增加付款说明

重点说明：

- 付费 ≠ 保证通过
- 付费 = 更快审核 + 固定曝光窗口

验收标准：

- 用户在提交前就知道可能会被拒绝
- 用户清楚知道付款后发生什么
- 用户知道缺哪些内容会卡审核

#### 3. Claim Listing Landing Page

目标：

- 提前验证工具方是否愿意留资

内容：

- Claim Listing 是什么
- 可以获得什么
- 为什么要认领
- 收集邮箱、公司、官网

CTA：

- `Claim This Listing`

验收标准：

- 页面能独立表达“认领”的价值
- 用户能直接留资
- 不需要先做复杂后台

#### 4. Stripe 与页面级埋点

统计：

- Pricing 访问
- Submit 访问
- Checkout 创建
- Payment Success
- Publish Success
- Tool Detail 点击
- Category 点击
- Guide → Tool 点击
- Comparison → Tool 点击
- Search 词
- Submit 来源

目标：

- 形成完整转化漏斗
- 找到用户在哪一步流失
- 看清哪些页面真的在推动下一步

验收标准：

- 管理端能看到漏斗
- 每一步都有可读的数量
- 能区分“想付费”和“真的付费”
- 能区分“浏览”与“推进到下一步”

新增模块：

- `Suitable For`
- `Not Suitable For`
- `Alternatives`
- `Pricing Snapshot`

- 让详情页从“介绍页”变成“决策页”

验收标准：

- 用户停留更久
- 用户更容易收藏、评论、点击官网
- 页面更像比较页，而不是卡片页

#### 3. 收费文案收紧

重点突出：

- 不是买广告位
- 是买更快上线
- 是买更快获得曝光
- 是买更容易被发现

验收标准：

- 用户不会把价格理解成“站点在硬收钱”
- 商业表达不压过产品表达

## 十一、什么情况下继续做，什么情况下先停

### 继续加内容

- 某个分类有搜索需求，但高质量页面太少
- 某篇 guide 能比现有页面更好地回答问题
- 某个 comparison 能明显帮助用户做决定

### 继续补工具

- 某个分类明显太薄
- 新工具和已有工具确实不同
- 能用一句话讲清楚这个工具的价值

### 先暂停扩张

- 网站开始显得乱
- 新页面和已有页面高度重复
- 工具增长速度超过了审核能力

### 归档或删除

- 工具已过时或被替代
- 官网失效
- 明确认定是重复条目
- 条目只会增加噪音

## 十二、最终判断标准

如果 60 天后能够达到：

- `150+` 工具
- `50+` Comparison
- `90+` Guide
- `10+` Claim Listing
- `5+` 付费工具
- 第一批真实收入

那么这个项目就不再只是一个工具目录，而是已经走到了“工具平台”的第一步。

如果做不到，也没关系。  
至少你会知道：

- 用户愿不愿意付费
- 哪类页面最能转化
- 哪类工具最值得继续做
- 你的站到底应该往哪边长
