# Weekly Operations Runbook

This runbook keeps AI Best Tool moving steadily without forcing a pivot or a major repositioning.

The goal is simple:

- keep the site healthy
- keep the directory fresh
- keep the content flowing
- keep monetization light and non-intrusive

## Operating Principles

- Prefer small, consistent updates over large rewrites.
- Prefer depth in a few categories over breadth everywhere.
- Prefer archive over delete when a tool is uncertain.
- Prefer editorial quality over volume.
- Prefer user trust over short-term monetization pressure.

## Weekly Rhythm

### Monday: Health Check

Spend 30 to 45 minutes reviewing the site and core systems.

- Check `/api/health`
- Check `/api/healthz`
- Check `/api/monitor` with token
- Review Vercel deployment status
- Confirm `/robots.txt` and `/sitemap.xml` are reachable
- Scan admin alerts and logs
- Confirm login, profile, submit, and comments still work

### Tuesday: Content Planning

Spend 30 minutes choosing the next content targets.

- Pick 1 to 3 high-intent guide pages
- Pick 1 to 3 comparison pages
- Pick 1 category or scene that needs more depth
- Review which pages already have enough tools to support them

### Wednesday: Tool Supply

Spend 45 to 60 minutes improving the directory itself.

- Review new candidates
- Promote qualified tools
- Archive stale tools that no longer belong in public listings
- Fix duplicates
- Fill missing metadata for promising tools
- Review categories that feel too empty

### Thursday: SEO and Indexing

Spend 30 to 45 minutes on discoverability.

- Check Search Console for indexing changes
- Inspect sitemap coverage
- Review top landing pages
- Look for pages with no impressions after publication
- Improve internal links where needed

### Friday: Product Polish

Spend 30 to 45 minutes tightening the user experience.

- Review homepage spacing and hierarchy
- Check mobile layout
- Inspect empty states
- Fix small UX issues from the week
- Make sure paid surfaces still feel natural

### Optional Weekend Block

Use this only if you have energy.

- Add one new high-quality content piece
- Clean up one category
- Improve one admin workflow
- Review community signals such as comments, favorites, and submissions

## Monthly Rhythm

### 1. Tool Database Review

- Review published tools for freshness
- Move stale tools to archived when needed
- Remove only confirmed dead or duplicate entries
- Re-score a sample of tools from the busiest categories

### 2. Content Review

- Check which guides are getting impressions
- Update titles or descriptions for weak pages
- Identify missing long-tail pages
- Add internal links from new pages to existing money pages

### 3. Growth Review

- Check organic traffic trend
- Review top queries
- Review top landing pages
- Confirm community actions are increasing
- Check whether the homepage still matches the brand direction

### 4. Monetization Review

- Review developer listing flow
- Check whether paid placement feels too aggressive
- Confirm any payment or contact flow is still working
- Review which surfaces can be monetized without harming trust

## Quarterly Rhythm

- Revisit category structure
- Decide whether any category should be merged, split, or retired
- Review the tool lifecycle policy
- Audit the homepage and top landing pages for drift
- Review site-wide copy for tone consistency
- Review whether the current content strategy still matches the business goal

## Decision Rules

### Add More Content When

- a category has search demand but too few quality pages
- a guide can answer a high-intent query better than existing pages
- a comparison page can help users choose faster
- internal linking can connect a new page to existing traffic

### Add More Tools When

- a category looks thin
- the new tools are meaningfully different
- a tool brings a new use case, not just a new brand name
- the tool can be described clearly in one sentence

### Pause Expansion When

- the site feels noisy
- the content no longer feels cohesive
- the new pages are too similar to existing ones
- the directory is growing faster than the review process

### Archive or Remove When

- the tool is stale or superseded
- the website is dead
- the listing is a confirmed duplicate
- the entry adds clutter instead of value

## Simple Weekly Checklist

- [ ] Health endpoints are okay
- [ ] Login and user flows are okay
- [ ] New content targets are selected
- [ ] New tools are reviewed
- [ ] Duplicates are cleaned up
- [ ] Search Console is checked
- [ ] One UX issue is fixed
- [ ] One growth signal is reviewed
- [ ] One monetization path is kept light and clear

## What Success Looks Like

- The site stays reliable.
- The directory gets slowly better every week.
- Users see useful additions without feeling overwhelmed.
- Search traffic grows from genuinely useful pages.
- Monetization exists, but does not dominate the experience.
