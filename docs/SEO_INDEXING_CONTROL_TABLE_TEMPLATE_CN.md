# 收录控制表模板

更新时间：2026-06-21

这是一份给运营和编辑日常使用的表，不用背规则。你只要按周更新这张表，就能决定：

- 哪些页面现在不能进 sitemap
- 哪些页面可以先留作内部流量页
- 哪些页面已经满足条件，可以进入 sitemap

核心原则：

> 先让页面有独立价值，再让它进入 sitemap。

## 一、怎么用

每周只做一次整理，建议固定在同一天。

### 第 1 步：先把新页面都记进来

凡是新建、改版、补内容、准备上线的页面，都先写进表里。

### 第 2 步：给每个页面一个当前状态

只有三种主状态：

- `不进 sitemap`
- `内部流量页`
- `可进 sitemap`

### 第 3 步：每周复查一次

如果页面变厚了、内链更强了、内容更稳定了，就把状态往前推一级。

## 三、状态判定规则

### 1. `不进 sitemap`

适合：

- 登录、注册、账户页
- 表单页、付款页、回调页
- 管理页、API 页
- 参数页、分页页
- 还没补厚的转化页

判断标准：

- 页面主要服务操作，不服务搜索
- 页面没有稳定独立主题
- 页面容易和别的页面重复

### 2. `内部流量页`

适合：

- 还在补内容的分类页
- 需要导流但不够厚的指南页
- `new`、`pricing`、`submit` 这类转化页

判断标准：

- 有用户价值
- 但暂时不值得和主内容页抢索引

### 3. `可进 sitemap`

适合：

- 首页
- 主入口页
- 内容成熟的分类页
- 内容成熟的指南页
- 代表性工具详情页

判断标准：

- 页面有独立搜索意图
- 页面内容够厚
- 页面能代表站点主题
- 页面有稳定的内部链接

---

## 四、升级规则

一个页面从“内部流量页”升级到“可进 sitemap”，至少满足以下 4 条中的 3 条：

1. 页面已经补齐标题、描述、正文和关键模块
2. 页面在站内有明显入口
3. 页面不是另一个页面的重复版
4. 页面有真实点击、收藏、评论、停留或外链信号

### 常见例子

- `pricing`：一般不升级，长期作为内部转化页
- `submit`：一般不升级，长期作为内部操作页
- `new`：如果只是“本周新增”入口，通常也不升级
- 分类页：如果工具数和内容都稳定，才升级
- guide 页：如果主题足够明确、内容足够深，才升级

---

## 五、降级规则

如果一个已经进 sitemap 的页面出现下面情况，就考虑降级：

- 页面内容太薄
- 访问量长期很低
- 主题开始和别的页面重复
- 页面不再适合作为公开内容页

降级动作通常是：

- 从 sitemap 移除
- 改成内部流量页
- 必要时加 `noindex`

---

## 六、推荐每周检查顺序

按这个顺序看，效率最高：

1. 首页、Guides、Explore 有没有还在正常导流
2. 分类页有没有补厚
3. 主力指南页有没有缺正文或缺内链
4. 代表性工具页有没有 logo、截图、标签、价格、评论
5. `new / pricing / submit / developer listing` 这类页有没有误进 sitemap

---

## 七、给你最实用的三个结论

- 想进 sitemap 的页面，先做成“内容页”
- 只做转化的页面，先留在内部流量层
- 不确定时，先不要进 sitemap，等下次复查
